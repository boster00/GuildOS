# Cat (Questmaster) reliability fix — 2026-04-28 overnight session

## TL;DR

Cat's sessions kept dying because:

1. The session would FINISH on cursor's side, the GuildOS DB still showed `idle`, and the cron nudger kept trying to followup against a corpse.
2. Cat's `system_prompt` (last edit 2026-04-15) predated the `items`/`item_comments` schema, the `judge_origin` mandate, the lockphrase gate contracts, and BCS/WWCD — so even when alive, Cat operated on a stale playbook.
3. With an idle queue and no firm anti-drift rule, Cat would invent off-task work (writing infra PRs) and then FINISH.

The fix is structural, not behavioral. Six commits on `main` between 00:23 and 01:05 local on 2026-04-28:

| commit | scope |
|---|---|
| `013a29b` | locked review loop (`questmaster.runReviewLoop`) + reconcile session lifecycle in cron + role-aware `.cursorrules` |
| `913370b` | `reconcileSessionLifecycle` only flags respawn for execute/escalated stage (not review/closing/purrview) |
| `fc14bd3` | `notifyQuestmaster` filters stranded purrview quests + points Cat at `runReviewLoop` |
| `dfabc51` | `guildmaster.respawnAdventurer` callable companion to the howTo |
| `947bd48` | cron auto-respawns Cat when EXPIRED with pending purrview work (whitelist + 24h cooldown) |
| `9db8b78` | skip Cat in `nudgeConfused` (her queue is purrview, not execute) |

Plus a DB-only change to Cat's `adventurers.system_prompt` (refreshed 2026-04-28; 2747 → 7404 chars) and a fresh cursor session via the locked spawn path.

## What changed concretely

### 1. `questmaster.runReviewLoop({questId})` — one-call locked review

Added to `libs/skill_book/questmaster/index.js`. Replaces the manual `confirmSubmission → judge → approve/bounce` orchestration Cat used to assemble herself. Internally:

- `questPurrview.confirmSubmission` (verifies SUBMIT lockphrase)
- For each item: `openai_images.judge` (gpt-4o, mandatory `judge_origin`)
- `questPurrview.approve` if all pass, else `questPurrview.bounce` with reason auto-built from judge rationales

Returns one of `{action: 'approved' | 'bounced' | 'manual_required' | 'confirm_failed' | 'approve_failed' | 'bounce_failed'}`. Cat never decides when to call the judge — the loop forces it. End-to-end smoke confirmed it on a fresh quest in <2 min (cron nudge → review stage with APPROVE lockphrase + per-item `judge_origin: openai_images.judge:gpt-4o`).

Manual `questmaster.reviewSubmission` howTo still exists for the non-image case.

### 2. `cron.reconcileSessionLifecycle` — auto-detect dead cursor sessions

Added to `libs/council/cron/index.js`, runs first on every 5-min cron tick (Vercel + local). Calls `cursor.syncSessionStatus` per adventurer with a `session_id`, normalizes DB `session_status` against upstream, warns when a non-dispatchable adventurer (EXPIRED / deleted) still has assigned active quests in `execute` or `escalated`.

For Cat specifically: the queue stage to check is `purrview`, not execute, AND if the upstream is EXPIRED with pending work, cron calls `guildmaster.respawnAdventurer` automatically. Circuit breaker: 24h cooldown, parsed from the backstory archive line that `respawnAdventurer` appends. Workers (CJGEO Dev, Researcher, etc.) stay manual-respawn — a human should decide whether the spawn prompt should change before recommissioning a worker.

### 3. `cron.notifyQuestmaster` — filter stranded quests + new nudge text

The old nudge surfaced every quest in `purrview` stage to Cat, including legacy quests that reached purrview via direct stage write (no SUBMIT lockphrase). Now: only quests with a `questExecution.submit_for_purrview` comment make the inbox; strands are logged and skipped. The nudge text now points Cat at `questmaster.runReviewLoop({questId})` as the canonical path, removing the old "if 90% satisfied" heuristic that predated the script-locked judge gate.

### 4. `cron.nudgeConfused` — exclude Cat

Cat's queue is purrview-stage, not execute-stage, so `deriveAdventurerStatuses` consistently marks her `confused` — which then triggered the generic worker nudge. Excluding Cat from `nudgeConfused` removes the dual-message confusion (`notifyQuestmaster` is the only path that nudges her now).

### 5. `guildmaster.respawnAdventurer({name, reason})` — callable companion

Added alongside the existing `toc.respawnAdventurer` howTo. Spawns via the locked `cursor.writeAgent` path (so the credentials block fires), updates `adventurers.session_id` + appends a backstory archive note, returns `{prior_session_id, new_session_id, url, upstream_status}`. The new agent reads its full `system_prompt` from the adventurers DB row at boot, so the canonical contract lives in one place.

### 6. Cat's `system_prompt` refresh (DB-only, no code diff)

The 2026-04-15 prompt told Cat to write to an `inventory[].comments` array (column dropped 2026-04-22), use the "90% satisfied" heuristic, invoke `claudecli` she doesn't carry, and didn't mention `judge_origin` / lockphrases / BCS / WWCD. Replaced with a tight 7.4KB prompt that:

- Pins Cat's loop to `searchQuests → runReviewLoop` with manual fallback for non-image items.
- Mandates `openai_images.judge` (gpt-4o) per item with `judge_origin` stamped on every verdict.
- References the items/item_comments schema correctly.
- Hard-bans code edits, PR opening, infra refactors. ("Refusing the temptation is the highest-leverage thing you can do here.")
- Tells Cat to STOP when the queue is empty (don't invent work; the prior session FINISHED after drifting into a cron-nudge re-verify PR exactly because of this).

### 7. `.cursorrules` Role-aware spawn section

Added near the top of GuildOS `.cursorrules`. If the spawn's system_prompt names you Cat / Questmaster, the rest of the file (no-new-files, billing, tech stack) does NOT apply — your contract lives in `adventurers.system_prompt` + the questmaster skill book; do not write product code or open PRs even if you spot a bug. Worker adventurers still get the full file.

## Operational verification

End-to-end smoke (live, not mocked) confirmed the entire pipeline on 2026-04-28:

- Created a fresh purrview quest with 2 image items.
- Cron's `notifyQuestmaster` filter recognized the SUBMIT lockphrase and nudged Cat.
- Cat (cursor cloud agent) executed `runReviewLoop`.
- Quest reached `review` in 44 seconds with the APPROVE lockphrase + per-item `judge_origin: openai_images.judge:gpt-4o`.

Then the overnight sanity loop in `scripts/_overnight-cat-sanity-loop.mjs` runs the same cycle every 30 min until 7am local, logging PASS/FAIL per cycle. Stable here = the cron→Cat→runReviewLoop pipeline holds through repeated invocations.

## Known limitations / not-yet-done

- **Non-image judging**: `runReviewLoop` returns `manual_required` for non-image items (text/JSON/doc). The manual `reviewSubmission` howTo handles those, but a future `claude-multimodal-read` integration could close the gap.
- **2 stranded review-stage quests** (`766fb022`, `63596f17`) reached `review` without going through Cat's gate (legacy direct stage write). Annotated with diagnostic comments via `guildmaster:stranded_diagnostic`. Proper fix needs a `housekeeping.resetStaleReview` weapon (out of scope tonight) — currently must be hand-fixed via SQL.
- **Workers** (`Researcher`, `CJGEO Dev`, etc.) are NOT on the auto-respawn whitelist. If their session EXPIREs, the cron warns but doesn't act — a human decides whether the spawn prompt should change first.
